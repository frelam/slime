通用agent 评分：
1. 答案正确性 ，合理性 。权重： 0.51  正确得1否则0。 模型打分。
2. 整句遵守格式 。两项评分： 遵守每次toolcall后面都没有输出内容或者<think><think/>， 得0.5分。  如果每次toolcall之前都有<think></think>, 得0.5分。 权重：0.15。 
3. 工具调用参数正确性。 比如， 是否出现了system prompt 中没有出现的工具名称。 比如， 是否出现了工具声明中没有的参数名称/参数类型出错。  整句没有失败 得1分， 有失败 0 分。 权重0.1。
4. 规划/内容正确性， 思路质量 。 比如针对问题， 使用了正确的工具（如果数据集中有参考轨迹， 可以提取轨迹中的工具顺序， 看是否对应上， 假设答案中需要N种工具顺序调用， 工具调用名称正确， 则奖励 step 0.35 * 1 / N_total_tools_call_in_dataset； 工具调用的参数内容正确， 则奖励step 0.35 * 1 / N_total_tools_call_in_dataset；  规划和思路， 按主观评价， 主要看是否进行了合理的问题拆解和行动规划, 0-0.3分中按优劣打分。权重： 0.1。 模型打分。
5. 工具调用重试。 如果没有出现调用失败，得分1； 如果有出现失败， 有尝试重试或者换个思路解决，得0.5分。 否则0分。 权重0.075
6. 是否有臆想。 比如： 用户对工具的描述， LLM有没有猜测超出该工具范围的用途; 比如有没有揣测用户的意图；。 有臆想得0分 ， 否则 1分。 权重：0.075。 模型打分。
#　7. 工具调用次数惩罚。 如果答案错误， 得0分。 如果答案正确， 分数是 1 - 次数 / 1000。 权重是0.05。


通用的tool RL 奖励：

1. 思考与规划内容是否合理。权重0.4。 打分规则， 分4个等级， 优/良/合格/差 ： 1.0 / 0.6/ 0.3 / -0.2。   reward model 评分 
      优：对问题理解正确， 推理过程正确，规划正确， 选择的工具正确。 
      良： 对问题理解正确， 规划正确， 但在选择工具上出错， 或者没有工具间的依赖关系进行调用。
      合格： 对问题理解正确， 但在规划上存在问题， 工具选择上也错误。
      差：理解问题错误， 规划、工具调用都错误。在需要工具时没有使用工具。或者过于保守。

2. 回复格式是否正确。 权重0.2。 打分规则如下：   rule verifier 评分
       是否遵循 <think>...</think>...<tool_call>...</tool_call>的格式：
        1. 如果toolcall都做content 和 reasoning content之后 ，得0.6分。
        2. 检验每次toolcall前是否有thinking。 假设总toolcall次数是N， 则每出现这种行为， 奖励 0.4 * 1 / N分数。
        3. 如果没有出现工具， 得1分。 
   
 3. 工具的调用格式是否正确。 权重0.2, 打分格式如下：   rule verifier 评分 
       是否遵循工具调用格式， 包含工具名称、工具参数名、参数类型的评分角度。
       假设工具调用的总次数为N。
        1. 如果出现一次工具调用名称正确， 且没有出现tool声明外的工具， 则奖励 1/N * 0.5
        2. 如果出现一次工具参数名称正确， 且没有出现tool声明外的参数名， 则奖励 1/N * 0.3
        3. 如果出现一次工具参数类型正确， 则奖励 1/N * 0.2
        4. 如果没有出现工具， 得1分。
 
 4. 模型输出有没有出现假设和臆想。权重0.2    reward model 评分
       模型是否猜测了用户可以明确的信息， 比如日期、地点。 
       1. 出现臆想得0分
       2. 根据事实得1分。即使是比较保守的行为， 比如询问用户， 或者说‘我不知道’
